1. Market and Context Target Market:  Tanzania. The platform must prioritize mobile-first performance above all else. Product Catalog:  Herbal supplements, powders, and tonics. Visual Strategy:  The UI must feature high-quality, culturally relevant, and inclusive imagery. The interactive design must degrade gracefully on slower network connections. Payment Infrastructure:  Credit cards are secondary. The system must integrate local mobile money gateways (M-Pesa, Tigo Pesa, Airtel Money) alongside standard global gateways. Compliance:  All product pages must include explicit health disclaimers, ingredient lists, and usage instructions. 2. Core Functional Requirements Feature Category Specific Requirement Critical Acceptance Criteria Product Discovery Fast, filterable product grid. Users can filter by benefit (e.g., energy, digestion) and form (powder, tonic). Performance Asset optimization for mobile networks. Heavy animations must disable automatically if the browser detects a slow connection. Images must lazy-load. Checkout Flow Frictionless, multi-step mobile checkout. The cart state must remain synchronized. Shipping calculates accurately based on local Tanzanian regions. Payments Mobile money API integration. The transaction must use an idempotency key. The system handles delayed webhook confirmations gracefully. Admin Controls Secure inventory and content management. The Code Preservation Prompt "When updating visual layouts or styling, you are strictly forbidden from altering or overwriting underlying functional logic, state management, or API connections. Modify only the presentation layer. If a visual change requires a structural logic change, you must ask for explicit permission first." 2. The Mobile-First Performance Prompt "Design the frontend architecture to aggressively prioritize mobile performance. Implement strict image optimization, lazy loading, and code splitting. Any heavy visual libraries or interactive animations must degrade gracefully or be conditionally loaded only for high-bandwidth connections." 3. The API Gateway Prompt "Design a strict API gateway. The frontend must never communicate directly with the database. All data requests must pass through a secure API layer. Define the exact JSON schema for every single request and response. If the payload does not match the schema exactly, the server must reject it immediately." 4. The CORS Policy Prompt "Configure strict Cross-Origin Resource Sharing policies on the backend. The backend must only accept requests from our specific production and staging frontend URLs. Deny all other origins by default. Do not use wildcard headers under any circumstances." 5. The State Synchronization Prompt "Map out the state synchronization strategy for the cart and checkout flow. The frontend must act as a visual representation of the backend state, not the source of truth. Detail how the frontend will handle network latency, failed mobile money API calls, and optimistic UI updates without ever showing incorrect totals to the user." 6. The Secret Management Prompt "Separate all environment variables into strict public and private categories. The frontend code must only have access to public, read-only keys. All write operations, mobile money processing keys, and database credentials must remain strictly locked on the backend server. Create a validation script that crashes the build if a private key is imported into the frontend bundle." 7. The Database Integrity Prompt "Set up the database schema with strict typing. You must use parameterized queries or a secure ORM for every single database interaction. Prevent all forms of injection. Write the transaction logic so that if any step of a payment process fails, the entire transaction automatically rolls back." 8. The Error Handling Prompt "Implement a global error handler. If an API route or database query fails, the server must log the detailed stack trace internally for debugging. The server must only return a generic, sanitized error message to the client. Never leak database schemas or internal logic in an API response."
